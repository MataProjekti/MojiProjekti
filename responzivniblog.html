Kako napraviti responzivan blog koristeći semantički HTML

U ovom postu naučićeš kako da napraviš jednostavan, čist i responzivan blog koristeći samo HTML i osnove CSS-a. Fokus je na semantičkim elementima koji Google i brovzeri mnogo bolje razumeju.

Semantički HTML znači koristiti elemente koji imaju smisao — kao što su <header>, <nav>, <article>, <section> i <footer>. Ovo čini tvoj kod čitljivijim i profesionalnijim.

1. Kako početi sa HTML-om

HTML je struktura svake web stranice i predstavlja kostur koji sve drži na okupu. Html je ono što korisnik vidi,CSS je za ulepšavanje korisničkog interfejsa dok je JavaSkript zaslužan za njegov rad Najvažnije za početak učenja htmla je da se nauči osnovni tagovi. Treba se fokusirati na ove tagove: p, h1-h6, img, a, div, section, article, header i footer.

Kada dobro razumeš HTML strukturu, kasnije ti CSS i JavaScript postanu mnogo lakši.

2. Osnovni JavaScript događaja

JavaScript omogućava da tvoja web stranica reaguje na akcije korisnika — to se zove "event". Najčešći eventi su: click, mouseenter, input, submit i keydown.

Na primer, kada klikneš na dugme — JavaScript može da pokrene funkciju koja menja tekst, boju, dodaje element ili šalje podatke.

3. Responzivni dizajn – Grid vs Flexbox

Responzivan dizajn znači da se stranica lepo prikazuje na mobilnom, tabletu i desktopu.

Flexbox je odličan za raspored u jednom pravcu — horizontalno ili vertikalno. Grid je bolji kada imaš složeniju strukturu sa više kolona i redova.

Oba sistema su laka kada ih jednom shvatiš, i koristićeš ih svakodnevno u frontend karijeri.
